# Knowledge Base

This directory contains the source-of-truth markdown files for RevX's RAG knowledge base.

## Structure

```
knowledge/
├── business/           # Company context, org structure, processes
│   ├── overview.md     # What Sanguine Bio does, key processes
│   ├── glossary.md     # Business & tech terms
│   └── team.md         # RevOps engineering team structure
│
├── salesforce/         # Platform knowledge
│   ├── architecture.md     # Orgs, pipeline, tech stack, integrations overview
│   ├── apex-patterns.md    # Trigger handler, service layer, selector, test patterns
│   ├── lwc-patterns.md     # Container/presentational, data patterns, events, styling
│   ├── governor-limits.md  # Limits quick reference, anti-patterns, monitoring
│   ├── experience-cloud.md # Portal architecture, components, security
│   ├── integrations.md     # n8n, Sevalla proxy, Platform Events, REST API
│   ├── permissions.md      # Profiles, permission sets, sharing, FLS, CRUD
│   └── schema/             # Auto-generated by schema crawler (rag crawl)
│
├── apps/               # Application-specific documentation
│   ├── case-gantt.md   # Kanban LWC for Case management
│   └── devops-tool.md  # SF DevOps CI/CD pipeline tool
│
└── runbooks/           # Operational procedures
    ├── deploy-to-prod.md       # Production deployment steps
    ├── sandbox-refresh.md      # Sandbox refresh procedure
    └── troubleshooting.md      # Common issues and fixes
```

## How It Works

1. Add or edit `.md` files in the appropriate subdirectory
2. Commit and push to trigger a container rebuild
3. On startup, the RAG sync script reads all files → chunks them → generates embeddings → stores in PostgreSQL
4. RevX and agents search this knowledge base semantically when they need context

## Schema Crawler

Auto-generate org schema docs from live Salesforce metadata:

```bash
# Crawl dev org, write files + push to RAG
rag crawl --org dev --output both

# Crawl prod org, only push to RAG (no files)
rag crawl --org prod --output rag

# Crawl specific objects
rag crawl --org dev --objects "Case,Account,Source_Change__c"
```

Schema crawler generates per-object docs with fields, record types, validation rules, and permissions.

## Writing Good Knowledge Docs

- **Use clear headers** (`##`, `###`) — the chunker splits on these
- **One topic per file** — helps categorization and retrieval
- **Be specific** — "The Account object has a custom field `Revenue_Tier__c`" beats "we have some custom fields"
- **Include context** — why something exists, not just what it is
- **Keep files under 5000 words** — very long files produce many chunks and dilute relevance

## Manual Operations

```bash
rag sync [--force]                              # Sync files to DB
rag crawl [--org dev] [--output both]           # Crawl SF org schema
rag search "query" [-k top_k] [-c category]     # Semantic search
rag add --title T --content C [-c category]     # Add dynamic knowledge
rag sources [-c category]                       # List all documents
rag stats                                       # Knowledge base stats
```
